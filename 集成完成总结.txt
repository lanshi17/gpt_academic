🚀 快速开始（3 步）
================================================================================

第1步：获取 API 密钥
  ✓ Linxi:       https://linxi.chat/          (注册 → 获取 API Key)
  ✓ SiliconFlow: https://cloud.siliconflow.cn/ (注册 → 申请 API Key)

第2步：配置 API 密钥

  方式 A - Docker 用户:
    编辑 docker-compose.yml，找到 environment 部分:
    ────────────────────────────────────────
    LINXI_API_KEY: "你的Linxi-API-Key"
    SILICONFLOW_API_KEY: "你的SiliconFlow-API-Key"
    ────────────────────────────────────────

  方式 B - 本地开发:
    编辑 config_private.py (约第 18-20 行):
    ────────────────────────────────────────
    LINXI_API_KEY = "你的Linxi-API-Key"
    SILICONFLOW_API_KEY = "你的SiliconFlow-API-Key"
    ────────────────────────────────────────

第3步：启动应用

  Docker 用户:
    $ docker-compose down
    $ docker-compose up -d
    $ docker-compose logs -f

  本地用户:
    $ python3 main.py

  验证：打开浏览器访问 http://localhost:12303
       在模型下拉菜单中应该看到新增的模型

🔍 问题排查
================================================================================

Q: 在模型列表中看不到新模型？
A: 1. 检查 config.py 或 config_private.py 中的 AVAIL_LLM_MODELS
   2. 确保 API 密钥配置正确
   3. 重启应用或 Docker 容器
   4. 清除浏览器缓存并刷新页面

Q: 出现 "API key is invalid" 错误？
A: 1. 确认 API Key 已正确复制（没有多余空格）
   2. 检查官网是否已生成该密钥
   3. 确认账户余额充足
   4. 尝试重新生成新的 API Key

Q: Docker 容器无法启动？
A: 1. 查看日志：docker-compose logs
   2. 检查 docker-compose.yml YAML 格式
   3. 检查端口 12303 是否被占用
   4. 尝试重新拉取镜像

Q: 如何选择 Linxi 还是 SiliconFlow？
A: • Linxi: 所有主流模型、无推理优化、国际模型更多
   • SiliconFlow: 推理模型专优、支持思维链、速度快
